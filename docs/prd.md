# 文本大数据知识抽取与图谱构建工具 PRD

## 概述 (Executive Summary)
本文档（PRD）详细描述了 **文本大数据知识抽取与图谱构建工具** 的功能需求。该平台旨在将多源文本数据集成管理、知识抽取、深度学习模型训练和知识图谱构建与可视化探索集于一体，为数据分析人员、标注人员、算法工程师和项目管理员等多种角色提供服务。我们设计的平台覆盖本体设计、知识抽取、数据映射、数据写入和图谱探索等流程【5†L51-L54】【5†L72-L77】，支持 Excel/CSV 等结构化数据映射和基于模型的三元组抽取。文档按模块逐页列出了各功能点，每个界面中手册截图上用红圈标出的按钮都在表格中逐一说明。文档还包括字段定义、接口说明、非功能需求、验收标准和扩展方向等内容。

## 1. 项目概述
我们构建的文本大数据知识抽取与图谱构建平台，致力于满足以下需求和目标：

- **多源文本数据管理**：支持导入并管理多种格式文本及结构化数据。
- **知识抽取与关系识别**：利用深度学习模型和大语言模型识别文本中的实体、关系、事件和属性。
- **深度学习模型训练**：提供语料管理、标注任务分配、标注工具、模型训练和效果评估流程。
- **知识图谱构建与管理**：支持基于本体设计导入 OWL/RDF、结构化/半结构化数据映射转换、图谱模型可视化编辑等。
- **图谱查询与探索**：提供基于图数据库的可视化查询、关系展开、路径分析等功能。

我们的目标用户包括系统管理员（负责用户、权限管理）、项目管理员（负责图谱项目和模型管理）、标注人员（进行实体关系标注）、算法工程师（训练抽取模型）和普通用户（进行图谱查询和分析）【5†L51-L54】。各角色通过基于角色的权限控制（RBAC）访问系统功能【6†L147-L150】，具体权限定义如下：

| 角色 | 权限说明 |
|---|---|
| 系统管理员 | 管理系统用户、角色和权限配置，维护系统基础数据。 |
| 项目管理员 | 创建/管理图谱项目与图谱模型，负责整体项目进度。 |
| 标注人员 | 执行数据标注任务，对文本进行实体和关系标注。 |
| 算法工程师 | 进行模型训练、优化及知识抽取工作。 |
| 普通用户 | 使用图谱查询、可视化和分析功能，查看知识图谱结果。 |

## 2. 功能架构
系统由**系统首页**、**平台管理**、**图谱项目管理**、**知识抽取与转化**、**图谱实例管理**五大模块组成，如下图所示：

```mermaid
flowchart LR
    A[系统首页(登陆后)] --> B[平台管理]
    B --> B1[用户管理]
    B --> B2[角色管理]
    B --> B3[修改密码]
    A --> C[图谱项目管理]
    C --> C1[知识图谱管理]
    C --> C2[图谱模型管理]
    A --> D[知识抽取与转化]
    D --> D1[半结构化数据转化]
    D --> D2[深度学习知识抽取]
    D --> D3[LLM知识抽取]
    A --> E[图谱实例管理]
    E --> E1[知识图谱探索]
```

- 平台管理模块（用户管理、角色管理、修改密码）用于系统用户及权限维护。
- 图谱项目管理模块包括**知识图谱管理**（创建/发布图谱项目）和**图谱模型管理**（创建/编辑本体模型与实体关系）。其中知识图谱管理支持定义项目名称、编码、存储配置等；图谱模型管理提供实体/关系编辑、OWL 文件导入导出、可视化等功能。
- 知识抽取与转化模块涵盖**半结构化数据转化**（如 Excel/CSV 转图谱）和**深度学习知识抽取**（语料管理、标注任务、模型训练、抽取执行）以及**基于LLM的知识抽取**。此架构参考典型知识图谱平台流程，支持结构化与非结构化数据的映射和三元组抽取【5†L72-L77】。
- 图谱实例管理模块包含**知识图谱探索**页面，支持图谱数据的可视化查询、关系展开和分析。

具体功能模块和流程如上，并通过图示的导航流程图呈现。

## 8. 非功能需求

### 8.1 性能要求
- 页面响应时间：关键操作页面加载时间小于3秒；图谱可视化加载时间小于5秒。
- 并发支持：系统能够支持100以上并发用户访问，同时处理后台模型训练任务。
- 数据量：能够处理百万级规模的节点和边。

### 8.2 安全要求
- **基于角色的权限控制（RBAC）**：采用 RBAC 模型为用户分配角色【6†L147-L150】，用户只能访问被授权的功能菜单和数据。
- **密码及敏感信息加密**：用户密码采用强哈希算法加密存储；关键数据通信使用 HTTPS，保证数据传输安全。
- **操作日志记录**：对用户的关键操作（如登录、配置变更、数据导出等）进行日志审计。
- **接口鉴权**：所有后台接口需进行身份验证和权限校验，防止非法访问。
- **文件上传校验**：对上传的文件进行类型、大小和内容扫描，防止上传恶意文件。

### 8.3 可用性要求
- 支持常见主流浏览器（Chrome、Edge）并适配常用屏幕分辨率。
- 界面友好，操作流程清晰，提示信息明确。
- 系统支持中文界面，交互符合中文用户习惯。
- 提供帮助文档或操作引导，降低用户使用门槛。
